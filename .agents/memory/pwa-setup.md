## Structure

- `public/manifest.json` — app manifest (start_url: /dashboard/dispatch, display: standalone)
- `public/sw.js` — hand-rolled service worker (no next-pwa library); cache-first for /_next/static/, network-first for navigation, offline fallback to /offline.html
- `public/offline.html` — static branded offline page
- `public/icons/` — icon-192.png, icon-512.png, apple-touch-icon.png (generated by Node.js pure PNG encoder using zlib; brand color #e05d2a)
- `src/app/sw-register.tsx` — "use client" component that calls navigator.serviceWorker.register('/sw.js')
- `src/app/layout.tsx` — exports `viewport` (Next.js 15 Viewport type) for theme-color/viewportFit; exports `metadata.manifest`; imports ServiceWorkerRegistration

## Key rules

**Why no next-pwa:** Avoided to prevent npm override conflicts (Replit blocks npm-run-path v6+). The manual SW is straightforward and fully controllable.

**SW scope:** /sw.js at public root → scope is /. The `Service-Worker-Allowed: /` header is set in next.config.ts headers() to allow the full scope.

**Cache-Control for sw.js:** Must be `max-age=0, must-revalidate` so the browser always checks for updates. Set in next.config.ts headers().

**Icon generation:** Icons are generated by a Node.js script using only zlib (built-in) — no npm packages needed. Creates solid-color PNG with a D lettermark. Script is not committed; re-run if icons need regeneration: the logic lives in the original implementation.

**Next.js 15 viewport:** The `viewport` export (separate from `metadata`) is required for theme-color and viewportFit. Using `metadata.themeColor` is deprecated in Next.js 14+.

## How to apply

- Any page that needs to be cached for offline: add its URL to PRECACHE_URLS in public/sw.js and bump CACHE_VERSION.
- To add a new app shortcut: add to `shortcuts` in public/manifest.json.
- To regenerate icons: run the Node.js zlib PNG script (see commit history for the generation code).
